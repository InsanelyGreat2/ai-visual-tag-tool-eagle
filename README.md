# AI视觉标签助手 Eagle 插件

`AI视觉标签助手` 是一个面向 Eagle 素材库的 AI 视觉标签辅助插件。它使用 Eagle 官方 AI 模型套件分析图片、动图、视频和可预览设计文件，并把结果限制在可控的标签池中，方便给游戏特效、视觉资产和参考素材批量补充标签。

## 功能

- 读取 Eagle 标签管理中的全部标签，或只读取指定 Eagle 标签组作为当前标签池。
- 默认只允许 AI 从当前标签池中选择标签；标签池外的结果会被过滤，不会写入素材。
- 可导入、管理和重置本地默认特效标签模板；模板只保存在插件本地，不会自动创建 Eagle 全局标签。
- 在“分析参数”的“分析阐述”中可开启“允许标签池外兜底建议”，让本地默认模板作为兜底候选参与分析；命中后仍需用户确认。
- 支持通过“追加当前选中”“替换为当前选中”和置顶采集条连续采集 Eagle 当前选中的素材，并自动去重。
- 支持图片、TGA、DDS、GIF、APNG、静态 WebP、动态 WebP、视频和可预览设计文件。
- 视频和动图通过 FFmpeg 抽帧后交给 Eagle AI 视觉模型分析；TGA/DDS 会通过 FFmpeg 转换为 PNG 后分析；静态 WebP 会按普通图片处理。
- 动态 WebP 在 FFmpeg 抽帧不稳定时会尝试使用 Eagle 缩略图或预览图兜底，并过滤 0KB 抽帧文件。
- 分析过程中显示整体进度、当前阶段、当前素材、待写入、已写入、失败和跳过统计。
- 写入标签时显示独立写入进度条。
- 暂停分析后，可立即对已经分析完成的素材点击“写入标签”；正在分析中的素材会自然结束，剩余未开始素材保持待处理。
- 支持单个素材重新分析、失败项重试、结果过滤、媒体预览、手动增删待写入标签和撤销上次写入。
- 全局 AI 提示词位于“AI 模型”板块，提供更大的预览和编辑区域。
- 默认不把素材标题传给 AI，避免文件名或标题误导画面识别；可在设置中手动开启。
- 可保存诊断抽帧、路径信息和 `diagnostic.json`，用于排查抽帧、路径和 AI 输入问题。

## 安装

1. 打开 Eagle 的插件管理。
2. 选择本地安装或导入本地项目。
3. 选择本插件目录。
4. 确认 Eagle 已安装并启用官方 AI 模型套件。
5. 如果需要分析视频、GIF、APNG、动态 WebP、TGA 或 DDS，确认 Eagle 的 FFmpeg 依赖可用。

## 使用方式

1. 在 Eagle 中选中要分析的素材。
2. 打开 `AI视觉标签助手`。
3. 点击“追加当前选中”或“替换为当前选中”把素材加入待分析列表。
4. 在标签池区域选择“全部 Eagle 标签”或某个标签组。
5. 如标签池为空，可点击“导入默认模板”作为本地候选标签。
6. 在“AI 模型”中检查 Eagle 默认视觉模型，并按需编辑全局 AI 提示词。
7. 在“分析参数”和“抽帧”中调整最大标签数、并发数、AI 重试次数、请求分块、抽帧频率和最大帧数。
8. 点击“开始分析”。
9. 在结果区确认标签，必要时手动增删或重新分析。
10. 点击“写入标签”把已勾选标签写回 Eagle 素材。

## 依赖

- Eagle
- Eagle 官方 AI 模型套件，并配置默认视觉模型
- Eagle FFmpeg 扩展，用于视频、GIF、APNG、动态 WebP 抽帧，TGA/DDS 转换和诊断帧保存

## 注意事项

- 本项目不包含任何命令行 AI 后端，也不会调用本地 CLI 分析素材。
- 插件不会自动创建 Eagle 全局标签。
- 删除标签池中的标签只会从当前插件候选池移除，不会删除 Eagle 标签管理中的真实标签。
- 默认模板兜底标签只有在开启“允许标签池外兜底建议”后才会参与分析，命中后仍需要确认和写入。
- 未开启兜底建议时，AI 返回的池外标签会显示在“已过滤”信息中，不会写入素材。
- 高置信标签默认也会进入待确认区；只有开启“自动写入高置信标签”后才会自动写回 Eagle。
- 动态 WebP 的抽帧能力取决于 Eagle FFmpeg 环境和源文件编码；插件已加入预览图兜底，但极端文件仍可能需要转码后再分析。
- TGA/DDS 识别依赖 Eagle FFmpeg 转换为 PNG；如果源文件使用 FFmpeg 不支持的特殊压缩格式，需要先转码为 PNG/JPG 后再分析。
- 高抽帧频率会增加分析时间和模型成本，长视频建议配合最大帧数限制使用。
